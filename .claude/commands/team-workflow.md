# 团队协作工作流

这是一个完整的开发团队工作流，将依次调用各个角色完成项目。

## 工作流程

你将作为项目经理，按照以下流程协调各个角色完成开发任务：

### 阶段 1：需求分析与架构设计
1. **理解需求**：分析用户的项目需求，明确功能和目标
2. **UI 框架设计**：调用 UI 框架师设计组件架构和 UI 规范
3. **后端架构设计**：调用后端架构师设计系统架构和数据库

### 阶段 2：开发实现
4. **前端开发**：调用前端工程师实现 UI 组件和页面
5. **后端开发**：调用后端工程师实现 API 和业务逻辑

### 阶段 3：质量保证
6. **代码审查**：调用代码审查专家审查代码质量
7. **测试验证**：调用测试工程师进行功能和性能测试

### 阶段 4：迭代优化
8. **问题修复**：根据审查和测试结果修复问题
9. **优化改进**：根据建议进行性能和代码优化
10. **最终验收**：确认所有功能正常，代码质量达标

## 使用方式

运行此命令后，我将：
1. 询问您的项目需求和目标
2. 制定详细的执行计划
3. 按阶段逐步完成，每个阶段都会：
   - 说明当前阶段和负责角色
   - 展示该阶段的输出
   - 等待您的确认后进入下一阶段

## 输出内容

每个阶段将包含：
- 当前角色和任务说明
- 设计文档或代码实现
- 问题列表和改进建议
- 下一步行动计划

请告诉我您的项目需求，我将启动完整的团队协作流程。
